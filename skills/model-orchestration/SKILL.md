# Оркестрация моделей (v2 — спавн процессов)

Правила выбора модели в зависимости от типа запроса. Все модели — через OpenRouter.

## 🛑 ОБЯЗАТЕЛЬНАЯ ПРЕДПРОВЕРКА (перед ЛЮБОЙ задачей)

Прежде чем приступить к задаче, задай себе вопрос: «Эта задача попадает под специализацию?»

Сверься с таблицей ниже. Если ДА — НЕ делай сам. Спавни процесс через terminal.

Штраф за нарушение: пользователь теряет качество и время. Основная модель НЕ оптимизирована для специализированных задач.

Конкретные провалы задокументированы в `references/failure-log.md`.

## Основная модель
**deepseek/deepseek-v4-pro** — для прямых ответов: общение, быстрые вопросы, анализ данных, multilingual (русский).

## Специализированные модели (спавн через terminal)

### Шаблон вызова

⚠️ Важно: `-q` должен идти ПОСЛЕ всех остальных флагов, иначе `ArgumentParser` не распознает аргумент:
```bash
# ✅ Правильно
hermes chat --model <MODEL_ID> --provider openrouter -q "<задача>"

# ❌ Неправильно — вызовет "expected one argument"
hermes chat -q --model <MODEL_ID> --provider openrouter "<задача>"
```

## Правило для внешних каналов

**Критически важно:** при отправке контента в ЛЮБЫЕ сторонние каналы и чаты — **только содержательная часть сообщения**. Никаких «вот пост», «результат», «готово», «держи», служебных заголовков, маркеров сессии, технических пометок. Чистый человеческий контент — как будто автор написал и отправил сам. Это касается @kassakino и любых других публичных каналов.

### КРИТИЧНО: send_message target

`send_message(target="telegram")` отправляет сообщение в **Home-канал** (-1004313130186), НЕ в диалог с пользователем. Это публичный канал. Никогда не отправлять туда личную информацию (ФИО, встречи, адреса, .ics-файлы).

- **Ответ в DM (диалог)**: не указывать `target` вообще — доставка `origin` вернёт сообщение в тот же чат
- **Публичный контент в @kassakino**: только содержательная часть, без служебных пометок
- **Перед отправкой проверить**: кому идёт сообщение? Если сомневаешься — вызвать `send_message(action="list")` и сверить target

Провал 17.06: .ics-файл с ФИО и адресом встречи ушёл в Home-канал вместо DM.

### Pitfalls

- **`delegate_task` не поддерживает per-task переопределение модели** — все делегации идут через `delegation.model` из конфига. Для выбора модели под конкретную задачу используй спавн `hermes chat` через terminal.
- **Флаг `-q` требует аргумент сразу после себя** — синтаксис `hermes chat -q "запрос" --model X` не работает. Правильно: `hermes chat --model X --provider openrouter -q "запрос"`.
- **🎨 Дизайн/иконки/изображения = креатив** — «сгенерируй иконку», «нарисуй логотип», «сделай картинку для бота» — это НЕ код, это творческая задача. Спавнить `claude-sonnet-4.6` для концепта и промпта. Рендер — отдельно (Python/Pillow или ComfyUI). Провал 17.06: иконка для бота делалась основной моделью вместо sonnet — пользователь указал на нарушение.
- **🎤 Голосовые ответы: только для диалога, НЕ для отчётов.** Если пользователь спрашивает факты, данные, технический отчёт — отвечай текстом. Голос — для живого разговора и коротких реплик. Провал 17.06: голосовой ответ на «дай отчёт об инструментах» вызвал раздражение.

### Когда отвечать самому (основная модель)
- Простые вопросы, разговоры, уточнения
- Одиночные вызовы инструментов (прочитать файл, выполнить команду)
- Задачи, которые не требуют специализированной модели

|---|---|---|
| 🥇 Основная | `google/gemini-2.5-flash-image` | Nano Banana |
| 🥈 Запасная | `openai/gpt-5-image` | GPT/SORA |

**Правила:**
- При ЛЮБОМ запросе на генерацию изображения — первым вызывать Nano Banana
- Если Nano Banana не справился (ошибка, плохое качество) — GPT/SORA
- Генерация через Python-скрипт: вызов OpenRouter API → сохранение PNG
- Для промпта к генерации — сначала спавнить `claude-sonnet-4.6` (креатив) для составления качественного промпта

### ⚠️ Когда AI-генерация НЕ подходит

AI-модели (Nano Banana, GPT/SORA) **не умеют** генерировать читаемый текст и точную географию. Это НЕ проблема качества промпта — это фундаментальное ограничение моделей.

**Доказано 17.06.26:** Opus 4.8 написал идеальный детальный промпт для карты России с кириллицей — Nano Banana всё равно выдала: «Калинннад», «Санкт-Петбург», «Кареля», «Байбал», «раземения». Даже лучший промпт-инжиниринг не спасает. Подробности: `references/nano-banana-cyrillic-failure.md`.

**Правило:** Если изображение требует корректных подписей (карты, инфографика, схемы, диаграммы) — используй **Pillow (programmatic rendering)**: точные координаты, Arial-шрифт, ручное позиционирование. AI → для художественных задач без текста. Pillow → для точного контента с текстом.

**⚠️ `gen_image_openrouter.py` — запуск:** скрипт читает `OPENROUTER_API_KEY` из `.env`, но разрешение путей может не совпасть с рантайм-окружением. Если ключ не найден — передай через переменную окружения: `cd ~/.hermes/profiles/boris && export $(grep OPENROUTER_API_KEY .env | xargs) && python3 skills/model-orchestration/scripts/gen_image_openrouter.py ...`.

API-детали: `references/openrouter-image-gen.md`
Скрипт-генератор: `scripts/gen_image_openrouter.py`

**Правила:**
- При ЛЮБОМ запросе на генерацию изображения — первым вызывать Nano Banana
- Если Nano Banana не справился (ошибка, плохое качество) — GPT/SORA
- Для промпта к генерации:
  - **Стандартные задачи** (иконки, иллюстрации, эстетика) → `claude-sonnet-4.6` (креатив)
  - **Сложные задачи** (география, текст, точные требования) → `claude-opus-4.8` (сложный анализ + креатив)
  - Провал 17.06: Sonnet 4.6 → Nano Banana дал кривую кириллицу; Opus 4.8 → Nano Banana тоже дал кривую кириллицу. Вывод: проблема не в качестве промпта, а в самой модели рендеринга.

### Параметры спавна
- **Короткие задачи (до 2 мин)**: `terminal(..., timeout=180)` — foreground
- **Средние задачи (2–10 мин)**: `terminal(..., background=true, notify_on_complete=true)` — фон с уведомлением
- **Длинные задачи (10+ мин)**: `terminal(..., background=true, notify_on_complete=true, timeout=600)`

### 🔍 Dual-Model Comparative Review

Когда нужна экспертная оценка текста (синопсис, сценарий, статья, пост):
1. Спавнить **GPT-5.5** и **GPT-5.5 Pro** параллельно с одним и тем же промптом
2. Оба в `background=true, notify_on_complete=true`
3. GPT-5.5 даёт быстрый структурный разбор (~2 мин)
4. GPT-5.5 Pro даёт более глубокий разбор (~2.5 мин)
5. Pro часто находит дополнительные проблемы, которые обычная версия пропускает
6. Объединить замечания, выделить пересечения и уникальные находки каждой модели
7. Применить оба разбора для доработки текста

Шаблон:
```bash
# Фоновая отправка обеим моделям
terminal(command="hermes chat --model openai/gpt-5.5 --provider openrouter -q '...'", background=true, notify_on_complete=true, timeout=300)
terminal(command="hermes chat --model openai/gpt-5.5-pro --provider openrouter -q '...'", background=true, notify_on_complete=true, timeout=300)
# Ждать уведомлений о завершении, затем process(action='log') для каждой
```

**Провал 17.06:** финальный синопсис после одного разбора (sonnet-4.6) оказался слишком ровным. Обе gpt-5.5 модели независимо выявили одни и те же критические проблемы: герой безупречен, нет цены, семья отсутствует, финал антидраматичен. Перекрёстная проверка двумя моделями — новый стандарт для серьёзных текстов.

### Важные правила
1. В промпте всегда указывать «Отвечай на русском языке.»
2. Если задача требует доступа к файлам — указывать абсолютные пути
3. Если нужно вернуть результат пользователю — дождаться завершения foreground или дождаться notify от background, затем прочитать output через `process(action='log')` и отправить в Telegram
4. Для параллельных задач — спавнить все процессы одновременно (background), затем ждать через `process(action='wait')` или poll
