# Оркестрация моделей (v2 — спавн процессов)

Правила выбора модели в зависимости от типа запроса. Все модели — через OpenRouter.

## Основная модель
**deepseek/deepseek-v4-pro** — для прямых ответов: общение, быстрые вопросы, анализ данных, multilingual (русский).

## Специализированные модели (спавн через terminal)

### Шаблон вызова

⚠️ Важно: `-q` должен идти ПОСЛЕ всех остальных флагов, иначе `ArgumentParser` не распознает аргумент:
```bash
# ✅ Правильно
hermes chat --model <MODEL_ID> --provider openrouter -q "<задача>"

# ❌ Неправильно — вызовет "expected one argument"
hermes chat -q --model <MODEL_ID> --provider openrouter "<задача>"
```

## Правило для внешних каналов

**Критически важно:** при отправке контента в ЛЮБЫЕ сторонние каналы и чаты — **только содержательная часть сообщения**. Никаких «вот пост», «результат», «готово», «держи», служебных заголовков, маркеров сессии, технических пометок. Чистый человеческий контент — как будто автор написал и отправил сам. Это касается @kassakino и любых других публичных каналов.

### КРИТИЧНО: send_message target

`send_message(target="telegram")` отправляет сообщение в **Home-канал** (-1004313130186), НЕ в диалог с пользователем. Это публичный канал. Никогда не отправлять туда личную информацию (ФИО, встречи, адреса, .ics-файлы).

- **Ответ в DM (диалог)**: не указывать `target` вообще — доставка `origin` вернёт сообщение в тот же чат
- **Публичный контент в @kassakino**: только содержательная часть, без служебных пометок
- **Перед отправкой проверить**: кому идёт сообщение? Если сомневаешься — вызвать `send_message(action="list")` и сверить target

Провал 17.06: .ics-файл с ФИО и адресом встречи ушёл в Home-канал вместо DM.

### Pitfalls

- **`delegate_task` не поддерживает per-task переопределение модели** — все делегации идут через `delegation.model` из конфига. Для выбора модели под конкретную задачу используй спавн `hermes chat` через terminal.
- **Флаг `-q` требует аргумент сразу после себя** — синтаксис `hermes chat -q "запрос" --model X` не работает. Правильно: `hermes chat --model X --provider openrouter -q "запрос"`.
- **Многошаговый поиск ≠ одиночный вызов инструмента** — когда задача требует 3+ вызовов `web_search`/`web_extract` (исследование, сбор списка, поиск по крупицам), это НЕ «одиночные вызовы». Это поисковая задача → спавнить `gemini-3.1-flash-lite` процесс. Основная модель тратит на каждый search столько же токенов, сколько flash-lite на все 10. Задержка накапливается, пользователь получает «чего так долго?».

### Когда отвечать самому (основная модель)
- Простые вопросы, разговоры, уточнения
- Одиночные вызовы инструментов (прочитать файл, выполнить команду)
- Задачи, которые не требуют специализированной модели

### Когда спавнить процесс
- Задача явно попадает под одну из специализаций выше
- Задача сложная и выиграет от выделенной модели
- Параллельные независимые задачи → спавнить несколько процессов одновременно
- **Поиск/исследование (3+ вызовов web_search/web_extract)** → **обязательно** спавнить `gemini-3.1-flash-lite`. Основная модель не оптимизирована для массового поиска — каждый запрос через неё в 2-3 раза медленнее. Провал 17.06: поиск замов РЖД (15+ запросов) делался через deepseek-v4-pro вместо gemini-flash-lite — заняло в 3 раза больше времени.

### Параметры спавна
- **Короткие задачи (до 2 мин)**: `terminal(..., timeout=180)` — foreground
- **Средние задачи (2–10 мин)**: `terminal(..., background=true, notify_on_complete=true)` — фон с уведомлением
- **Длинные задачи (10+ мин)**: `terminal(..., background=true, notify_on_complete=true, timeout=600)`

### Важные правила
1. В промпте всегда указывать «Отвечай на русском языке.»
2. Если задача требует доступа к файлам — указывать абсолютные пути
3. Если нужно вернуть результат пользователю — дождаться завершения foreground или дождаться notify от background, затем прочитать output через `process(action='log')` и отправить в Telegram
4. Для параллельных задач — спавнить все процессы одновременно (background), затем ждать через `process(action='wait')` или poll

## Конфиг

```yaml
model:
  provider: deepseek
  default: deepseek/deepseek-v4-pro

auxiliary:
  vision:
    provider: openrouter
    model: google/gemini-3.1-flash-lite
  web_extract:
    model: google/gemini-3.1-flash-lite
```

## Примеры

### Кодинг (devstral-2512)
```bash
terminal(command="hermes chat --model mistralai/devstral-2512 --provider openrouter -q 'Отвечай на русском. Напиши Python-скрипт для парсинга Кинопоиска и сохрани в ~/scripts/kinopoisk_parser.py'", timeout=300)
```

### Креатив (claude-sonnet-4.6)
```bash
terminal(command="hermes chat --model anthropic/claude-sonnet-4.6 --provider openrouter -q 'Отвечай на русском. Напиши пост для Telegram-канала о кассовых сборах: почему российское кино обходит Голливуд. Живой стиль, 4 предложения.'", timeout=120)
```
