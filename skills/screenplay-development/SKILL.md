# Screenplay Development

Сквозной пайплайн разработки киносценариев: от идеи до финального документа.

## Workflow

### 1. Исследование (OSINT)

Собрать все доступные данные о прототипе/теме:
- `web_search` + `web_extract` по ключевым источникам
- Биографии, интервью, некрологи, архивные материалы
- Сохранить ключевые факты, цитаты, даты

### 2. Синопсис — первый проход

- **Модель:** `claude-sonnet-4.6` (креатив)
- В промпте: контекст героя, жанр, хронометраж, структура
- Выход: литературный синопсис, живой текст без техразметки
- Формат: сплошной текст 4500–6000 знаков, без маркеров «Акт I»
- Начинать с сильного крючка (детство, поворотный момент)

### 3. Антагонист и конфликт

- **Модель:** `claude-sonnet-4.6` (креатив)
- Антагонист должен быть ЛИЧНЫМ: имя, биография, мотивация, своя правда
- Три уровня препятствий: техническое, организационное, психологическое
- Кульминационная сцена на пике сюжета
- Антагонист не карикатурный злодей — у него своя правда

### 4. Синопсис — инъекция антагониста

Вшить антагониста и препятствия в синопсис:
- Первое появление — как можно раньше
- Препятствия распределены по хронологии
- Кульминация вшита в финальный акт

### 5. Humanizer

- **Навык:** `humanizer` (29 AI-паттернов)
- Дополнительно: `references/russian-literary-humanization.md` (7 русскоязычных паттернов)
- Сломать ровный ритм абзацев
- Убрать декларативную мораль («Фильм о том, как...»)
- Добавить беспорядка: неполные предложения, «не знаю», «а может, и нет»
- Убрать rule-of-three (если их больше 2 на текст)
- Заменить «show, don't tell»: вместо «это не спорт» → показать сценой

### 6. Dual-Model Review

- **Модели:** `gpt-5.5` + `gpt-5.5-pro` параллельно (см. `model-orchestration`)
- Обе получают один промпт: профессиональный редакторский разбор
- Требования: честность, прямота, сильные/слабые стороны, структурные дыры
- Сравнить оба разбора: выделить пересечения и уникальные находки
- Pro версия часто находит дополнительные проблемы

### 7. Финальная доработка

Применить все замечания из reviews:
- Дать герою внутренний недостаток
- Добавить цену (семья, деньги, карьера)
- Сделать соратников конфликтными партнёрами
- Обосновать название символически
- Подготовить финальную сцену через setup/payoff в ранних актах
- Убрать «красивую немоту» — добавить прямые разговоры
- Показать конкретные провалы вместо «тысячи часов»

### 8. PDF

- Шрифт: Courier New, 14pt (сценарный стандарт)
- Полный скрипт: `scripts/gen_pdf_courier.py`
- fpdf2 с TTF-шрифтом (Courier New), имя шрифта отличное от встроенного Courier
- Встроенный Courier — Latin-1, без кириллицы. Всегда использовать TTF с новым именем.

## Примеры промптов

### Синопсис (claude-sonnet-4.6)
```
Отвечай на русском языке. Ты профессиональный сценарист.
Напиши СИНОПСИС полнометражного фильма [название].
Жанр: [жанр]. Хронометраж: ~110 минут.
[Контекст героя, ключевые события, антагонист]
ТРЕБОВАНИЯ: художественный текст, без техразметки, без таблиц, 4500–6000 знаков, сплошной текст, начинать с детства.
```

### Рецензия (gpt-5.5)
```
Отвечай на русском языке. Ты — опытный сценарист и редактор киностудии.
Прочитай синопсис [название] и дай профессиональный разбор.
Не пересказывай — анализируй.
[Текст синопсиса]
Разбери: сильные стороны, слабые, что работает на зрителя, что нет, структурные проблемы, драматургические дыры. Будь честным и прямым.
```

## Pitfalls

- **Герой без изъяна** — самая частая ошибка. Герой должен сомневаться, ошибаться, платить цену.
- **Антагонист-функция** — если антагонист просто «мешает», он не работает. Нужна биография, мотивация, своя правда.
- **Название без обоснования** — если фильм называется «Як-130», зритель должен понять почему именно этот самолёт, а не любой другой.
- **Слишком ровный финал** — «тихая победа без оваций» работает только если есть замыкающий жест (звонок, книга, фраза).
- **Декларативность** — фразы «это фильм о...» не должны быть в тексте. Зритель дойдёт сам.
